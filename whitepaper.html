GrainVDB Whitepaper

Topological Manifold Intelligence on Apple Silicon Unified Memory

1. Abstract

The current state of manifold resolution is dominated by cloud-first architectures that impose a "Latency Tax" of 50-200ms per probe. On-device solutions often rely on generic CPU-bound libraries (e.g., OpenBLAS) that fail to leverage the specific architectural advantages of Apple M-series silicon. GrainVDB introduces a Direct Metal Access (DMA) engine combined with Topology Audit Logic to achieve sub-ms local resolutions with zero context loss.

2. Metal-Core: Unified Memory Optimization

Traditional vector DBs move data between memory spaces. GrainVDB utilizes Unified Memory to provide zero-copy access to the GPU. Our vectorized kernels are written in 16-bit half precision, doubling the effective memory bandwidth.

λ = Σ (q_i * d_{offset+i}) on MPS

By bypassing the PCIe bottleneck and utilizing the SoC's integrated GPU, GrainVDB achieves a sustained 16x speedup over standard CPU baselines.

3. Middle-Out™: Quasicrystal Phase Folding

Standard Product Quantization (PQ) is destructive; it clusters manifolds and loses the "topological shape" of the data. GrainVDB employs 5-fold Quasicrystal (Penrose) Phase Folding. We map 128-dimensional manifolds into a phase-fringe interference space.

Instead of floating-point coordinates, we store the Interference Wrap Counts. This achieves 16x density while ensuring that the topological relationships required for Topology Audits remain intact in the folded state.

4. Audit Logic: The End of Hallucination

Flat manifold resolution is context-blind. It retrieves the "average" nearest state, leading to context-mixing (e.g., merging "Apple" the fruit and "Apple" the company). GrainVDB implements a Topological Audit Filter.

We compute the Çech Coboundary Energy (δ) across top-k results. If the results do not "glue" topologically, the system flags a "Regime Fracture."

E_{gluing} = ||δs||²

This allows for "Zero-Hallucination RAG" where the system refuses to answer if the retrieved data is logically inconsistent.

5. Conclusion

GrainVDB is not just a database; it is a math-first engine for the next generation of privacy-focused, local-first AI applications on macOS and iOS.
